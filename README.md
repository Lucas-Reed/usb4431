# USB-4431 触发后区间平均长漂监测

面向 NI USB-4431 的本地桌面监测程序。采集进程连续、同步读取 AI0–AI3；指定通道出现上升沿后，对触发后的 `(起点, 终点]` 区间分别计算四路平均电压，并将每次触发生成的四个结果显示为长期趋势点。

## 已实现功能

- NI-DAQmx 连续四通道采集，默认 DC 耦合、±10 V、IEPE 关闭。
- 独立采集进程，包含 8 秒启动超时、停止排空、异常断开及未完成窗口报告。
- 触发阈值、回差、最小间隔和启动高电平抑制。
- 精确的 `(起点, 终点]` 样本边界；起点为 0 时排除触发样本。
- 跨采样块、单块多触发及任意重叠计算窗口。
- 参数运行期间锁定；停止时拒绝新触发并完成已建立窗口。
- 四张独立 Canvas 趋势图，支持触发序号、触发时间、运行时间三种横轴。
- 最新一次触发窗口的 AI0–AI3 原始采样波形；采集块到达后由浏览器逐帧刷新，可悬停查看每个点的时间和电压。
- 滚轮缩放、拖动平移及右键重置缩放。
- 每通道最新值、相对首点漂移、最小值、最大值。
- 完整结果保存在 Python 内存；图表均匀抽取最多 5,000 个代表点。
- 原始采样点仅保留当前/最新触发窗口，下一次触发时替换，避免长时间运行产生不可控的内存占用。
- 单通道和全通道 CSV；全通道导出后标记当前数据已保存。
- 清空和退出前检查未导出结果。
- 无硬件模拟模式，包含慢漂、噪声和 12 ms 触发脉冲。

## 环境要求

- Windows 10/11 x64
- Python 3.12
- NI-DAQmx 驱动（仅硬件模式需要）
- USB-4431（仅硬件模式需要）

## 开发运行

```powershell
python -m pip install -e ".[dev]"
python -m usb4431_monitor
```

程序默认选择模拟模式，无需 NI 硬件即可验证完整流程。

## 平均窗口定义

设触发样本序号为 `t`，实际采样率为 `fs`，窗口为 `(start, end]`，则：

```text
first = t + floor(start × fs) + 1
last  = t + floor(end × fs)
count = last - first + 1
```

例如实际采样率 10,000 S/s、窗口 20–120 ms 时，使用触发后的第 201 至 1,200 个样本，共 1,000 点。

## 硬件验收建议

软件自动测试不替代真实 USB-4431 验收。发布前应按需求使用至少 1,000 个宽度不小于 10 ms 的触发脉冲，并使用恒定电压及已知波形与离线结果逐项比对。设备名称默认为 `Dev1`，也可在界面刷新并选择 NI MAX 中的设备名。
